# AI-Based-Security-Alert-Prioritization-for-SOC-Operations
## Project Overview
Security Operations Centers (SOCs) receive thousands of alerts every day. Manually investigating every alert can lead to analyst fatigue and delayed response to real threats.
This project uses Machine Learning to automatically prioritize security alerts based on their characteristics and historical old data incident classifications. The objective is to help SOC analysts focus on the most important alerts first.

## Problem Statement

SOC teams often face alert overload due to the large volume of security events generated by monitoring tools.
Challenges include:
* Alert fatigue
* Delayed incident response
* Large numbers of false positives 
* Limited analyst resources
This project aims to reduce analyst workload by automatically prioritizing alerts using AI.

## Initial Approach
The project ws initially developed with a synthetic dataset generated using Python libraries such as NumPy and Pandas.
Random security-related attributes were created, including:
* Failed Login Attempts
* Malware Score
* Asset Criticality
* Privileged User Status
A Random Forest model was trained on this generated dataset to understand the complete machine learning workflow.
Although the model worked successfully, the dataset could not represent real SOC environments because the data was synthetically generated and did not accurately reflect real-world SOC alert behavior.

## Improvement to Real SOC Data
To make the project more realistic and aligned with SOC operations, the synthetic dataset was replaced with Microsoft's GUIDE cybersecurity dataset.The dataset contains real security alert information such as:
* Alert Category
* Suspicion Level
* Last Verdict
* Incident Grade
These attributes were used to train the AI model for alert prioritization.

## Project Workflow
### Step 1: Data Collection - 
Loaded the Microsoft GUIDE security alert dataset.
### Step 2: Data Preprocessing -
* Selected relevant columns
* Handled missing values
* Converted categorical data into numerical values
### Step 3: Priority Mapping -
Incident grades were mapped to alert priorities according to criteria in uploaded dataset:
* TruePositive → Critical
* BenignPositive → Medium
* FalsePositive → Low
### Step 4: Model Training -
A Random Forest Classifier was trained using:
* Category
* Suspicion Level
* Last Verdict
### Step 5: Evaluation - 
Model performance was evaluated using prediction accuracy. During experimentation with the initial synthetic dataset, the model achieved an accuracy of approximately 94%.
### Step 6: Alert Prioritization - 
The trained model predicts the priority of new security alerts and helps analysts focus on critical incidents.
